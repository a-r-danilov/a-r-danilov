# Проект «Gastro Hub»: управление данными ресторанной сети


## Описание проекта

**Gastro Hub** — сеть заведений общественного питания в центре Москвы (рестораны, бары, пиццерии, кофейни).  

**Задача:** создать БД для оптимизации управления бизнесом:  
- структурировать данные о заведениях, меню, менеджерах, продажах;  
- реализовать аналитические запросы для принятия решений.

## Данные и инструменты

- **Исходные данные:** дамп в форматах `.sql` и `.zip`.  
- **СУБД:** PostgreSQL.  
- **Клиент:** DBeaver (для восстановления дампа и работы с БД).  

## Структура БД

Созданы схемы и таблицы:  
1. **`cafe.restaurant_type`** (enum) — типы заведений: `coffee_shop`, `restaurant`, `bar`, `pizzeria`.  
2. **`cafe.restaurants`** — информация о заведениях:  
   - `restaurant_uuid` (PK, UUID),  
   - название, тип, меню.  
3. **`cafe.managers`** — данные о менеджерах:  
   - `manager_uuid` (PK, UUID),  
   - имя, телефон.  
4. **`cafe.restaurant_manager_work_dates`** — периоды работы менеджеров:  
   - составной PK (`restaurant_uuid`, `manager_uuid`),  
   - даты начала и окончания работы.  
5. **`cafe.sales`** — данные о продажах:  
   - составной PK (`date`, `restaurant_uuid`),  
   - средний чек (`avg_check`).

## Аналитические задачи

### Задание 1. Топ‑3 заведения по среднему чеку (в разрезе типов)
- **Цель:** выявить лучшие заведения для премирования менеджеров.  
- **Результат:** таблица с топ‑3 заведениями для каждого типа (`ресторан`, `бар` и т. д.), средний чек округлён до 2 знаков.


### Задание 2. Динамика среднего чека по годам (материализованное представление)
- **Цель:** отследить изменения среднего чека за годы (кроме 2023).  
- **Результат:** таблица с колонками:  
  - год,  
  - название заведения,  
  - тип заведения,  
  - средний чек за год и предыдущий год,  
  - изменение в %.

### Задание 3. Топ‑3 заведения по частоте смены менеджеров
- **Цель:** выявить заведения с высокой текучестью менеджеров.  
- **Результат:** список из 3 заведений с количеством смен менеджеров.

### Задание 4. Пиццерия с самым большим меню
- **Цель:** найти пиццерии с максимальным числом позиций в меню.  
- **Результат:** таблица с названиями пиццерий и количеством пицц (если несколько — вывести все).


### Задание 5. Самая дорогая пицца в каждой пиццерии
- **Цель:** проанализировать ценовую политику меню.  
- **Результат:** таблица с колонками:  
  - название заведения,  
  - тип блюда (`Пицца`),  
  - название пиццы,  
  - цена.

### Задание 6. Обновление цен на капучино
- **Цель:** увеличить цены на капучино на 20% с блокировкой изменений на время операции.  
- **Решение:** транзакция с `LOCK TABLE` для предотвращения конфликтов.  
- **Комментарий:** в скрипте пояснено использование блокировки и обновления.


### Задание 7. Единый формат номеров менеджеров
- **Цель:** привести номера менеджеров к единому формату: `8-800-2500-***` (порядковый номер по алфавиту, начиная с 100).  
- **Решение:**  
  - хранение старого и нового номера в массиве (`[новый, старый]`);  
  - блокировка таблицы `managers` на запись (доступ на чтение сохранён).


## Результат

- Нормализованная схема БД `cafe` с комментариями к структуре.  
- Скрипт для:  
  - создания таблиц и enum;  
  - заполнения данных из дампа;  
  - реализации представлений и аналитических запросов.  
- Документация по логике обновлений (задания 6–7).
