# Проект «Врум‑Бум»: аналитика продаж автомобилей

## Описание


Проект для сети автосалонов «Врум‑Бум» по структурированию и анализу данных о продажах автомобилей за период **2015–2023 гг.**

**Проблема**  
Компания столкнулась с трудностями управления данными из‑за:
- неструктурированного формата сырых данных;
- роста сети салонов и объёма информации о продажах, поставках, запасах и клиентах.


**Цель**  
1. Нормализовать данные и загрузить их в PostgreSQL.
2. Реализовать набор аналитических запросов для поддержки бизнес‑решений.


## Задачи и решения

### 1. Структурирование данных

- Создана схема `raw_data` с таблицей `sales` для первичной загрузки сырых данных из CSV.
- Разработана нормализованная схема `car_shop` (до 3NF) с отдельными таблицами для:
  - брендов;
  - моделей;
  - цветов (с учётом связи «многие‑ко‑многим»);
  - продаж;
  - клиентов и др.
- Для каждого поля выбран оптимальный тип данных с комментариями (например, `price NUMERIC(9,2)` для точности расчётов).
- Первичные ключи реализованы с автоинкрементом (`SERIAL`), настроены внешние ключи и ограничения (`NOT NULL`, `CONSTRAINTS`).

### 2. Аналитические запросы

Реализовано 6 ключевых запросов для бизнес‑аналитики:

1. **Процент моделей без параметра `gasoline_consumption`**  
   - Оценивает полноту данных о расходе топлива.

2. **Средняя цена брендов по годам (с учётом скидки)**  
   - Группировка по бренду и году, округление до 2 знаков после запятой.
   - Результат сортируется по бренду и году.

3. **Средняя цена по месяцам за 2022 год (с учётом скидки)**  
   - Разбивка по месяцам, сортировка по возрастанию.
   - Цена округляется до 2 знаков после запятой.

4. **Список купленных машин по пользователям**  
   - Использование `STRING_AGG` для объединения моделей в строку.
   - Формат: «Бренд Модель» (например, «Tesla Model 3»).
   - Сортировка по имени пользователя.

5. **Максимальная и минимальная цена по странам (без скидки)**  
   - Агрегация по стране происхождения бренда.
   - Отдельные колонки для макс. и мин. цены.

6. **Количество клиентов из США**  
   - Фильтрация по коду телефона `+1`.
   - Возврат единственного значения — количества пользователей.

## Технологии

- **PostgreSQL** — основная СУБД для хранения и обработки данных.
- **DBeaver / psql** — инструменты для загрузки данных и выполнения запросов.
- **SQL** — язык для нормализации, загрузки и аналитики.
- **CSV** — формат входных данных.

## Результат

- Нормализованная схема БД `car_shop` с полной документацией типов данных.
- Единый SQL‑скрипт, включающий:
  - создание и заполнение таблиц;
  - выполнение всех аналитических запросов.
- Обоснование выбранной структуры БД (в комментариях к полям и отдельной документации).
