# Проект «Бибип»: аналог СУБД на Python


## Суть проекта


Создание **упрощенног аналога СУБД** на Python:
- работа с файлами как с таблицами базы данных;
- реализация CRUD‑операции (создание, чтение, обновление, удаление);
- использование индексов для быстрого доступа к данным.

**Цель:** понять, как устроена внутренняя работа реальных СУБД.

## Технические требования

- **Язык:** Python (версия ≥ 3.11, рекомендуется 3.13).  
- **Хранение данных:** текстовые файлы (`*.txt`).  
- **Индексы:** отдельные файлы с колонками `id` и номер строки в основном файле.  
- **Ограничения:**  
  - нельзя загружать весь файл данных в память;  
  - индексы можно загружать в память (они малы).  

## Структура данных

Основные файлы:
- `cars.txt` / `cars_index.txt` — данные об автомобилях и их индексы.  
- `models.txt` / `models_index.txt` — модели автомобилей и их индексы.  
- `sales.txt` / `sales_index.txt` — записи о продажах и их индексы.  

**Формат строк:** фиксированная длина (500 символов) + `\n`.  
**Ключевое поле:** `vin` (уникальный идентификатор автомобиля).


## Задания


### 1. Сохранение автомобилей и моделей
- **Задача:** добавить запись о машине и модели в соответствующие файлы.  
- **Результат:** обновление `cars.txt`, `cars_index.txt`, `models.txt`, `models_index.txt`.  


### 2. Сохранение продаж
- **Задача:** записать информацию о продаже в `sales.txt` и `sales_index.txt`.  
- **Дополнительно:** обновить статус машины в `cars.txt` на `sold`.  


### 3. Вывод доступных машин
- **Задача:** показать список машин со статусом `available`.  
- **Требования:** сортировка по `vin`.  


### 4. Детальная информация о машине
- **Задача:** вывести данные о машине по `vin`:  
  - `vin`, `model_name`, `model_brand`, `price`, `date_start`, `status`, `sales_date`, `sales_cost`.  
- **Особенность:** поля `sales_date` и `sales_cost` могут быть `None` (если машина не продана).  


### 5. Обновление VIN‑кода
- **Задача:** исправить ошибочный `vin` у машины.  
- **Важно:** перестроить индекс после изменения ключевого поля.  


### 6. Удаление продажи
- **Задача:** удалить запись о продаже из `sales.txt` и обновить статус машины на `available`.  


### 7. Топ‑3 самых продаваемых моделей
- **Задача:** вывести топ‑3 моделей по количеству продаж.  
- **Критерии сортировки:**  
  1. Количество продаж (`sales_count`).  
  2. Цена (если количество продаж равно).  
- **Поля в результате:** `car_model_name`, `brand`, `sales_count`.


### Результат

Рабочий аналог СУБД на Python с поддержкой:
- хранения данных в файлах;
- индексов для быстрого поиска;
- CRUD‑операций.
