# Проект: интеграция данных и построение витрины по заказчикам для маркетплейса товаров ручной работы


## Основная задача

Интегрировать данные из нового внешнего источника в существующее хранилище данных (DWH) и построить **инкрементальную витрину** с отчётом по заказчикам (`dwh.customer_report_datamart`).

Витрина необходима для:
- внедрения программы лояльности;
- изучения интересов пользователей;
- улучшения системы рекомендаций товаров;
- корректировки маркетингового плана по акциям.

## Что нужно сделать

1. **Подключить и изучить новый источник данных**  
   - Источник расположен в схеме `external_source`.
   - Содержит две таблицы:  
     - `craft_products_orders` (данные по мастерам, товарам и заказам);
     - `customers` (данные по заказчикам).


2. **Загрузить данные в хранилище**  
   - Перенести данные из `external_source` в схему `dwh`.
   - Разместить данные в соответствующих таблицах измерений (`d_craftsman`, `d_customer`, `d_product`) и фактов (`f_order`).
   - Дополнить существующий скрипт загрузки данных, чтобы интеграция выполнялась инкрементально (с учётом регулярных обновлений из источника).


3. **Спроектировать витрину по заказчикам**  
   - Создать DDL для витрины `dwh.customer_report_datamart`.
   - Включить в витрину ключевые метрики (см. ниже).
   - Обеспечить инкрементальный расчёт данных (с учётом периода загрузки).

4. **Реализовать скрипт обновления витрины**  
   - Написать SQL‑скрипт для инкрементального расчёта показателей витрины.
   - Использовать логику, аналогичную существующей витрине по мастерам (`craftsman_report_datamart`).


## Ключевые метрики витрины

Витрина должна содержать:
- идентификатор заказчика и его персональные данные (Ф. И. О., адрес, дата рождения, email);
- финансовые показатели (сумма трат, доход платформы, средняя стоимость заказа);
- статистику заказов (количество, статусы, время выполнения);
- аналитические данные (популярная категория товаров, популярный мастер);
- отчётный период (год и месяц).


## Исходная структура данных

- **Схема `dwh`** — хранилище с таблицами измерений и фактов, а также существующими витринами.  
- **Схемы `source1`, `source2`, `source3`** — текущие источники данных.  
- **Схема `external_source`** — новый источник для интеграции.


## Ожидаемый результат


1. Данные из `external_source` успешно загружены в `dwh` и соответствуют структуре хранилища.  
2. Создана витрина `dwh.customer_report_datamart` с полным набором требуемых метрик.  
3. Реализован скрипт для инкрементального обновления витрины.  
4. Витрина готова к использованию для аналитики и принятия бизнес‑решений.
